## [1942. The Number of the Smallest Unoccupied Chair](https://leetcode.com/problems/the-number-of-the-smallest-unoccupied-chair/)

To solve this problem, we'll explore 2 possible approaches:

1. Brute Force
2. Priority Queues

Regardless of the approach used in this article, the first step is to sort the `times` array by arrival times, but before sorting `times`, we need to add each friend's ID to the corresponding entry in the array. This is necessary because sorting the array changes the order of the entries, and we need a way to keep track of which friend corresponds to each set of arrival and departure times.. 

Also note that, in as much as we have infinite chairs, we only care about the number of chairs enough to serve our friends, which is equal to the size of the times array.

### **Approach 1: Brute Force**

In this approach, we'll create an array chairs with the same size as our input times, which will keep track of the next available time for each chair. All values in chairs will be initialized to `0` because all chairs are initially empty. Now we iterate through the times; in each iteration, we iterate through the chairs array to find a chair where `chairs[j] <= current friend's arrival time`. Once we find that chair, `j` is the seat the friend will take, and we set `chairs[j]` to the current friend's departure time because that's when it becomes available again.

Example 1:

> `times = [[1,4],[2,3],[4,6]], targetFriend = 2`
>
> After adding each friend's ID to each time, `times` becomes: `[[1,4,0],[2,3,1],[4,6,2]]`
>
> `times` is already sorted based on arrival times.
>
> `chairs = [0,0,0]`
>
> ```
> for i = 0, arrival time = 1, departure time = 4, friend = i; iterating through chairs:
> 	j = 0, chair[j] = 0 and it's <= arrival_time so it's available, we set chair[j] = 
> 	4 (departure time) and break out of the loop.
> 	
> for i = 1, arrival time = 2 departure time = 3, friend = i; iterating through chairs:
> 	j = 0, chair[j] = 4, not valid, continue.
> 	j = 1, chair[j] = 0, and it's <= arrival_time so it's available, we set chair[i] 		= 4 (departure time) and break out of the loop.
> 	
> for i = 2, arrival time = 4 departure time = 6, friend = i; iterating through chairs:
> 	j = 0, chair[j] = 4, and it's <= arrival_time so it's available, we set chair[j] 		= 6 (departure time) and break out of the loop.
> ```
>
> As we can see our `targetFriend` will make use of chair `0`.

**Algorithm**

1. Initialize a new variable `n` to the size of the `times` array.

2. Append the friend ID to each entry in the `times` array.

3. Sort the `times` array based on arrival times.

4. Initialize a new array called `chairs` with a length equal to `n`, filling it with zeros.

   Loop through the `times` array using an index `i` from `0` to `n - 1`:

   - Retrieve `arrival_time`, `departure_time`, and `friend` from `times[i]`.
   - Loop through the `chairs` array using an index `j` from `0` to  `n - 1`:
     - If `chairs[j]` is less than or equal to `arrival_time` and `friend` matches `targetFriend`, return `j` immediately. If not, update `chairs[j]` to `departure_time`.

### **Implementation**

```python
class Solution:
    def smallestChair(self, times: List[List[int]], targetFriend: int) -> int:
        n = len(times)
        times = [[times[i][0], times[i][1], i] for i in range(n)]
        times.sort(key=lambda x:x[0])
        
        chairs = [0] * n

        for i in range(n):
            for j in range(n):
                if chairs[j] <= arrival_time:
                    if friend == targetFriend:
                        return j
                    chairs[j] = departure_time
                    break
```

**Time and Space Complexity Analysis**

##### **Time Complexity**:

- Sorting the `times` list: `times.sort()` sorts the list based on arrival times. This takes `O(n log n)`, where `n` is the number of friends.

- Iteration Breakdown:

  - The outer loop runs `n` times, once for each friend.
  - The inner loop iterates through the `chairs` array, which can run up to `n` times for each iteration of the outer loop in the worst case.

  Thus, the combined time complexity of the nested loops is `O(n×n) = O(n^2)`

Therefore the overall **Time Complexity** is `O(n^2)` since `O(n^2)` dominates ` O(n log n)`,

The **Space Complexity** is `O(n)` due to the additional `times` and `chairs` arrays, both of which can hold up to `n` elements.



### Approach 2: Priority Queues

In this approach, we’ll make use of two priority queues: `free_chairs` and `occupied_chairs`. The `free_chairs` queue will track the available chairs, while `occupied_chairs` will manage the currently occupied ones. Here's why we chose priority queues:

- **`free_chairs`**: Since the problem requires assigning the smallest available chair number to each arriving friend, a priority queue allows us to efficiently prioritize and select the chair with the smallest number.
- **`occupied_chairs`**: This queue helps us prioritize friends by their departure times. Once a friend leaves, we can efficiently free up their chair at the exact time they depart, ensuring chairs are available for future arrivals. This queue will store a pair (departure time, chair).

By using priority queues, we can efficiently retrieve the smallest available chair and free up the soonest available seat, optimizing both arrival and departure handling in the process.

Example 1:

> `times = [[3,10],[1,5],[2,6]], targetFriend = 0`
>
> After adding each friend's ID to each time, `times` becomes: `[[3,10,0],[1,5,1],[2,6,2]]`
>
> After sorting `times`: based on arrival times `[[1,5,1],[2,6,2],[3,10,0]]`
>
> As observed, all friends arrive before any of them depart, which means each friend will occupy a chair in sequence, that is:
>
> - friend `1` takes chair `0`
> - friend `2` takes chair `1`
> - friend `0` takes chair `2`

Example 2:

> `times = [[1,4],[2,3],[4,6]], targetFriend = 2`
>
> After adding each friend's ID to each time, `times` becomes: `[[1,4,0],[2,3,1],[4,6,2]]`
>
> `times` is already sorted based on arrival times.
>
> ```
> initially,
> 	free_chairs = [0,1,2]
> 	occupied_chairs = None
> 
> At time 1, Friend 0 arrives and takes the smallest availble free chair (0):
> 	free_chairs = [1,2]
> 	occupied_chairs = [(4,0)]
> 	
> At time 2, Friend 1 arrives and takes the smallest availble free chair (1):
> 	free_chairs = [2]
> 	occupied_chairs = [(4,0),(3,1)]
> 
> At time 3, Friend 1 departs which makes chair 1 free again:
> 	free_chairs = [1,2]
> 	occupied_chairs = [(4,0)]
> 
> At time 4, Friend 0 departs which makes chair 0 free again:
>     free_chairs = [0,1,2]
> 	occupied_chairs = None
> 
> At the same time 4, Friend 2 arrives and takes the smallest availble free chair (0):
> 	free_chairs = [1,2]
> 	occupied_chairs = [(6,0)]
> 
> At time 5 nothing happens
> 
> At time 6, Friend 2 departs, which restores our free_chairs and occupied_chairs to how it was at the beginning.  
> ```
>
> As you can see, out `targetFriend` will make use of chair `0`.

##### **Algorithm**

1. Initialize a new variable `n` to the size of the `times` array.

2. Append the friend ID to each entry in the `times` array.

3. Sort the `times` array based on arrival times.

4. Initialize Priority Queues:

   - Create `free_chairs`, initialized with chair numbers from `0` to `n - 1`.

   - Create an empty `occupied_chairs` priority queue.

5. Initialize a new variable `idx` to `0` to track the next arriving friend.

6. Iterate Through Time:

   - Loop from the earliest arrival time to the latest departure time:
     - Free Departing Chairs:
       - Check if any friends are departing at the current time. If so, free their chairs by removing them from `occupied_chairs`.
     - Process Arrivals:
       - If `idx < n` and there is a friend arriving at the current time:
         - Retrieve the smallest free chair.
         - If this friend is the target friend, return the chair immediately.
         - Otherwise, assign the chair to the friend, adding their departure time and chair number to `occupied_chairs`.
         - Increment `idx` to process the next arriving friend.

##### **Implementation**

```python
class Solution:
    def smallestChair(self, times: List[List[int]], targetFriend: int) -> int:
        n = len(times)
        times = [[times[i][0], times[i][1], i] for i in range(n)]
        times.sort(key=lambda x:x[0])

        free_chairs = [i for i in range(n)]
        occupied_chairs = []

        idx = 0
        for time in range(times[0][0], times[-1][1]):
            while occupied_chairs and occupied_chairs[0][0] == time:
                leaving, chair =  heapq.heappop(occupied_chairs)
                heapq.heappush(free_chairs,chair)
                
            if idx < n and times[idx][0] == time:
                arrival, leaving, friend = times[idx]
                chair = heapq.heappop(free_chairs)
                
                if friend == targetFriend:
                    return chair
                
                heapq.heappush(occupied_chairs, (leaving, chair))
                
                idx += 1

```

**Time and Space Complexity Analysis**

###### **Time Complexity:**

- Sorting the `times` list: `times.sort()` sorts the list based on arrival times. This takes **O(n log n)**, where `n` is the number of friends.

- Main loop over `time`:

  - The loop iterates over all time units between the earliest arrival (`times[0][0]`) and the latest departure (`times[-1][1]`). Let’s denote this time range as `T` where T = `latest departure - earliest arrival`

  - Each iteration checks and processes the friends arriving and leaving, which involves:
    - Removing from `occupied_chairs` or `free_chairs` (using `heapq.heappop`): **O(log n)**.
    - Adding to `occupied_chairs` or `free_chairs` (using `heapq.heappush`): **O(log n)**.

​	Therefore, for each time unit, heap operations are performed, so this part will take **O(T log n)**.

Thus, the overall **Time Complexity** will be `O(nlogn + Tlogn)`

**Space Complexity:**

- `free_chairs` list: This list stores the indices of free chairs. It has a size of **O(n)**, where `n` is the number of friends.
- `occupied_chairs` heap: This heap is used to store the occupied chairs and their corresponding leaving times. In the worst case (just like in the first example), all friends occupy a chair at the same time, so this can also grow to a size of **O(n)**.
- `times` list: A new list `times` is created to hold the arrival and leaving times along with the friend's index. This list has a size of **O(n)**.

Thus, the overall **Space Complexity** will be `O(n)`

### **Further Optimization**

Currently, the algorithm iterates through every time from the earliest arrival to the latest departure, which can be inefficient if the times are well spaced. We can optimize this by jumping to the next relevant time based on the following conditions:

1. **Determine Next Useful Time**:
   - If Friends are Arriving and Chairs are Occupied:
     - Set `time` to the minimum of the next friend's arrival time and the earliest departure time from `occupied_chairs`.
   - If No Chairs are Occupied:
     - Set `time` to the next friend's arrival time.
   - If No More Friends are Expected:
     - Set `time` to the earliest departure time from `occupied_chairs`.
2. **Exit Condition**:
   - If none of the above conditions apply, break out of the loop, as all friends have arrived and departed.

##### **Implementation**

```python
class Solution:
    def smallestChair(self, times: List[List[int]], targetFriend: int) -> int:
        n = len(times)
        times = [[times[i][0], times[i][1], i] for i in range(n)]
        times.sort(key=lambda x:x[0])

        free_chairs = [i for i in range(n)]
        occupied_chairs = []


        idx = 0
        time = times[0][0]
        while True:
            while occupied_chairs and occupied_chairs[0][0] == time:
                leaving, chair =  heapq.heappop(occupied_chairs)
                heapq.heappush(free_chairs,chair)
                
            if idx < n and times[idx][0] == time:
                arrival, leaving, friend = times[idx]
                chair = heapq.heappop(free_chairs)

                if friend == targetFriend:
                    return chair
                heapq.heappush(occupied_chairs, (leaving, chair))
                idx += 1
            
            if idx < n and occupied_chairs:
                time = min(times[idx][0], occupied_chairs[0][0])
            elif idx < n:
                time = times[idx][0]
            elif occupied_chairs:
                time = occupied_chairs[0][0]
            else:
                break

```

##### **Time and Space Complexity Analysis**

###### **Time Complexity**:

- Sorting the `times` list: This part is the same as the first version: `O(n log n)`.
- Main loop advancing to meaningful times:
  - Instead of iterating through each unit of time, the loop advances only to meaningful time points:
    - The next friend arrival (`times[idx][0]`).
    - The next friend departure (`occupied_chairs[0][0]`).
  - In the worst case, there are **2n meaningful events** (each friend arriving and leaving at distinct times).
  - For each event, you are performing heap operations (`heappop` and `heappush`), both of which take `O(log n)`.

Thus, the overall **Time Complexity** will be `O(nlogn)`

The **Space Complexity** remains the same as the first.
