## **[2938. Separate Black and White Balls](https://leetcode.com/problems/separate-black-and-white-balls/)**

### Approach 1: Counting

In this problem, we need to find the minimum number of adjacent swaps required to sort the balls in such a way that all the `0`s are moved to the front and all the `1`s are pushed to the back. For example, given the list `balls = [1, 1, 0]`, we can't just swap the first and last ball directly to get `balls = [0, 1, 1]` in one step, because they are not adjacent. So, we have to swap them step by step, only between adjacent balls.

Let’s walk through an example to understand how this works:

1. Start with `balls = [1, 1, 0]`.
   First, we swap the adjacent `1` and `0`:
   - Swap `balls[1]` and `balls[2]`, so now `balls = [1, 0, 1]`, and the number of steps is `1`.
2. Now, we swap the adjacent `1` and `0` again:
   - Swap `balls[0]` and `balls[1]`, so now `balls = [0, 1, 1]`, and the number of steps is `2`.

So in total, it took us `2` steps to separate the balls.

Instead of physically performing each swap, we can just count how many swaps would be needed without actually changing the list.

The key observation is that for each `0`, the number of swaps needed depends on how many `1`s have been seen before it. Each `1` in front of a `0` needs to be swapped with that `0`.

Let’s break down the example again `balls = [1, 1, 0]`:

- The first ball is `1`, so we set `previous_ones = 1`.
- The second ball is `1` again, so we update `previous_ones = 2`.
- The third ball is `0`, so the number of swaps needed is `steps += 1 (representing the current zero) * previous_ones`, which simplifies to `steps += previous_ones`. 

Thus the required steps is still `2`

#### **Algorithm**

1. Initialize `n` as the length of the input string.
2. Initialize `ans` to 0 to store the minimum steps required.
3. Initialize `previous_ones` to 0 to track the number of ones seen so far.
4. Iterate through each character in the string (index `i` from 0 to `n-1`):
   - If the character at index `i` is `"1"`, increment `previous_ones` by 1.
   - If the character is `"0"`, add `previous_ones` to `ans`.
5. Return `ans` as the final result.

#### **Implementation**

```python
class Solution:
    def minimumSteps(self, s: str) -> int:
        n = len(s)
        prev_ones = 0
        ans = 0
      
        for i in range(n):
            if s[i] == "1":
                prev_ones += 1
            else:
                ans += prev_ones
        
        return ans
```

**Complexity Analysis**

- **Time Complexity**: O(n) The algorithm iterates through the input string once, where n is the length of the string. Each operation within the loop (comparison, addition, and assignment) takes constant time. Therefore, the total time complexity is linear, O(n).

- **Space Complexity**: O(1) The algorithm uses only a few variables (`n`, `ans`, `previous_ones`) regardless of the input size. No additional data structures that scale with input size are used. Thus, the space complexity is constant, O(1).
