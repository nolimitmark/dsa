## [921. Minimum Add to Make Parentheses Valid](https://leetcode.com/problems/minimum-add-to-make-parentheses-valid/)

To solve this problem, we will explore 2 possible approaches:

1. Using a Stack (Extra Memory)
2. Using Counters (No Extra Memory)

### **Approach 1: Stack-Based Solution**

This approach uses a stack data structure to track and match parentheses. The core idea is to remove matching pairs of parentheses (`()`) as we encounter them, leaving only the unmatched ones that need correction.

For example, if our stack contains `["(", "(", "("]` and we encounter a closing parenthesis `)`, we can pop the last opening parenthesis since it forms a matching pair. After processing the entire string, any remaining parentheses in the stack represent those that need matching pairs to become valid.

Just like in our example above, after the pop operation, our stack is left with  `["(", "("]` now we know we need two more closing parenthesis to form matching pairs, which means we will have a total of `stack_length * 2` if we assume we add our closing parenthesis. So now our answer will be `(stack_length  * 2) - stack_length` which is simply just the number of items left in our stack.

##### **Algorithm**

1. Initialize an empty stack

2. For each character in the input string:

   - If the stack is empty, push the current character

   - Otherwise:
     - If the stack's top character is `(` and the current character is `)`, pop from the stack
     - Otherwise, push the current character

3. Return the stack's length

##### **Implementation**

```python
class Solution:
    def minAddToMakeValid(self, s: str) -> int:
        ans = 0
        stack = []

        for par in s:
            if stack:
                if stack[-1] == "(" and par == ")":
                    stack.pop()
                else:
                    stack.append(par)
            else:
                stack.append(par)
        
        return len(stack)
```

##### **Time and Space Complexity Analysis**

The **Time Complexity** of the program will be `O(n)`, while the **Space Complexity** will also be `O(n)` because imagine a scenario (worse case) where the input consists of only `(` or only `)` , we will have to add every item to our stack. where `n` is the length of the input.

#### **Approach 2: Counter-Based Solution**

This approach is similar to the first approach, but in this we'll simulate the stack using two variables, one to keep track of the number of `(` and the other to keep track of the number of `)` we've encountered so far rather than using an actual stack.

We'll do this by checking in every iteration, if the current item is `(`, just increment the `open` variable by 1 else, we will first check if `open` is greater than 0, if that's the case we will just decrement it by 1 (stack pop simulation) because we assume with the current `)` it forms a valid pair. But if that's not the case, we increment our `close` variable by 1.

Finally our `open` and `close` variables will hold the remaining number of `(` or `)` that were not able to be matched so the answer will be `((open + close) * 2) - (open + close)` which is simply just `open + close`.

##### **Algorithm**

1. Initialize two counters: `opening` and `closing` to 0
2. For each character in the input string:
   - If it's an opening parenthesis `(`, increment `opening`
   - If it's a closing parenthesis `)`:
     - If there's an unmatched opening parenthesis (`opening > 0`), decrement `opening`
     - Otherwise, increment `closing`
3. Return the sum of both counters

##### **Implementation**

```python
class Solution:
    def minAddToMakeValid(self, s: str) -> int:
        opening, closing = 0, 0
        
        for par in s:
            if par == "(":
                opening += 1
            else:
                if opening > 0:
                    opening -= 1
                else:
                    closing += 1
                    
        return opening + closing
```

##### **Time and Space Complexity Analysis**

The **Time Complexity** of the program will be `O(n)` where `n` is the length of the input, while the **Space Complexity** will be `O(1)` because no significant extra memory was used aside a 2 variables.
