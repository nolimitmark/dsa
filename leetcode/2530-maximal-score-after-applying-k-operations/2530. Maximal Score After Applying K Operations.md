## [2530. Maximal Score After Applying K Operations](https://leetcode.com/problems/maximal-score-after-applying-k-operations/)

To solve this problem, we'll expore two possible approaches:

1. Brute Force
2. Min Heap (Priority Queue)

### **Approach 1: Brute Force**

This problems requires us to maximize our answer after `k` operations, meaning we have to be `*greedy*` in each operation and select the largest number from our `nums` array. To achieve this, in each operation, we'll iterate through our `nums` array, select the largest number from it, and add it to our answer.

**Algorithm**

1. Initialize a variable `n` to the length of the input array `nums`.

2. Initialize a variable `ans` to 0, which will store the result to be returned.

3. Iterate `k` times, and in each iteration:

   - Initialize two variables, `maxVal` and `maxValIndex`, both to zero. These will store the maximum value and its corresponding index.

   - Iterate through `nums`, and update `maxVal` and `maxValIndex` to the current value and the current value index if the current value is greater than the `maxVal`.

   - Add `maxVal` to `ans`.

   - Divide `maxVal` by 3, round it up using `math.ceil()` and replace the value at index `maxValIndex` in `nums` with `maxVal`.

4. Return `ans`, which is the final accumulated sum of the maximum values after `k` iterations.

**Implementation**

```python
class Solution:
    def maxKelements(self, nums: List[int], k: int) -> int:
        n = len(nums)
        ans = 0
        
        for i in range(k):
            maxVal, maxValIndex = 0, 0
            for j in range(n):
                if nums[j] > maxVal:
                    maxVal, maxValIndex = nums[j], j
            ans += maxVal
            nums[maxValIndex] = ceil(maxVal / 3)
        
        return ans
```

**Time and Space Complexity Analysis**

- **Time Complexity**: The time complexity of the algorithm is `O(k * n)`. This is because for each of the `k` iterations, we traverse the array `nums` of length `n` to find the maximum value. Thus, the total time taken is proportional to the product of `k` (number of iterations) and `n` (length of the array).
- **Space Complexity**: The space complexity is `O(1)`, meaning constant space. The algorithm uses only a fixed amount of extra space for variables like `maxVal`, `maxValIndex`, and `ans`. No additional data structures or memory allocations grow with the size of the input array, which keeps the space usage minimal.

### **Approach 1: Min Heap (Priority Queue)**

This approach improves upon the brute-force method by reducing the time complexity at the cost of some additional space. Instead of iterating through the `nums` array on every operation, we can leverage a Min Heap (Priority Queue) to efficiently retrieve the largest number in `O(1)` time.

Since a Min Heap prioritizes the smallest values, we can negate each number before storing it in the heap to simulate a Max Heap, where the largest value is prioritized. Given that the problem guarantees all values in `nums` are positive (with a minimum value of 1), we don’t need to worry about negative numbers interfering with the logic.

**Algorithm**

1. Initialize a variable `ans` to 0 to store the result.
2. Negate all the values in `nums` to prepare for using a Min Heap as a Max Heap.
3. Heapify the modified `nums` to convert it into a Min Heap.
4. Iterate `k` times, and in each iteration:
   - Pop the largest (negated) value from the heap and store its absolute value in `maxVal`.
   - Add `maxVal` to `ans`.
   - Divide `maxVal` by 3, round it up using `math.ceil()`, and negate it again.
   - Push the negated value back into the heap.
5. Return the final value of `ans`.

**Implementation**

```python
class Solution:
    def maxKelements(self, nums: List[int], k: int) -> int:
        ans = 0
        
        nums = [-num for num in nums]
   
        heapq.heapify(nums)

        for i in range(k):
            maxVal =  abs(heapq.heappop(nums))
            ans += maxVal
            maxVal = ceil(maxVal / 3)
            heapq.heappush(nums, -maxVal)

        return ans
```

**Time and Space Complexity Analysis**

- **Time Complexity**: The time complexity of the algorithm is `O(k * log n)`. This is because:
  - The `heapq.heapify(nums)` operation takes `O(n)` time to initially build the heap from the list.
  - For each of the `k` iterations, retrieving the maximum value (`heapq.heappop`) and reinserting a modified value (`heapq.heappush`) both take `O(log n)` time due to the heap operations.
  - Therefore, the total time complexity is dominated by the `k` heap operations, resulting in `O(k * log n)`.
- **Space Complexity**: The space complexity is `O(n)`. You are storing all elements of the input array `nums` in the heap, which takes up `O(n)` space. Additionally, the algorithm uses a few extra variables (`ans`, `maxVal`), but their space is negligible compared to the heap, which requires space proportional to the input size.
