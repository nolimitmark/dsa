## [962. Maximum Width Ramp](https://leetcode.com/problems/maximum-width-ramp/)

To solve this problem, we'll explore two possible approaches:

1. Using Brute Force
2. Using a Monotonic Stack

### **Approach 1 (Brute Force)**

For each element in the array, we search for the rightmost element that's greater than or equal to it. The maximum difference between such indices will be our answer.

Let's analyze the example array: `nums = [6,0,8,2,1,5]`

> For each index, let's find if there's a larger or equal number to its right:
>
> ```
> Index 0 (value 6):
>   - Looking right: [0,8,2,1,5]
>   - First larger/equal number: 8 at index 2
>   - Ramp width: 2 - 0 = 2
> 
> Index 1 (value 0):
>   - Looking right: [8,2,1,5]
>   - First larger/equal number: 8 at index 2
>   - Also: 2 at index 3, 1 at index 4, 5 at index 5
>   - Largest ramp width: 5 - 1 = 4
> 
> Index 2 (value 8):
>   - Looking right: [2,1,5]
>   - No larger/equal numbers
>   - No ramp possible
> 
> Index 3 (value 2):
>   - Looking right: [1,5]
>   - First larger/equal number: 5 at index 5
>   - Ramp width: 5 - 3 = 2
> 
> Index 4 (value 1):
>   - Looking right: [5]
>   - First larger/equal number: 5 at index 5
>   - Ramp width: 5 - 4 = 1
> 
> Index 5 (value 5):
>   - No numbers to the right
>   - No ramp possible
> ```

The maximum width among all these ramps is 4, occurring between index 1 (value 0) and index 5 (value 5).

##### **Algorithm** 

1. Initialize `ans = 0` to track the maximum ramp width
2. For each index `i` from 0 to `n - 1`:
   - For each index `j` from `i + 1` down to `n - 1`:
     - If `nums[j] >= nums[i]`, update `ans = max(ans, j - i)`
3. Return `ans`

##### **Implementation**

```python
class Solution:
    def maxWidthRamp(self, nums: List[int]) -> int:
        ans = 0
        
        for i in range(len(nums)):
            for j in range(i + 1, len(nums)):
                if nums[j] >= nums[i]:
                    ans = max(ans, j - i)
        return ans
```

##### **Time and Space Complexity Analysis**

The **Time complexity** of the program will be `O(n^2)` because it uses a nested loop where the outer loop iterates `n` times and the inner loop at most will iterate `n - 1` times, where `n` is the length of the input array. While the **Space Complexity** will be `O(1)` because no significant extra memory was used aside the `ans` variable.



### **Approach 2: Monotonic Stack**

Instead of comparing every possible pair of numbers like in the brute force approach, we can be smarter about which combinations we need to check. This approach uses extra memory to achieve better time efficiency.

The solution works in two main phases:

1. Building the Stack (Left to Right)

   - We create a stack that only keeps track of potential "starting points" for our ramps
   - These starting points are numbers that could potentially give us the maximum width
   - We only store numbers that are smaller than what we've seen before (creating a decreasing sequence)

2. Finding Maximum Widths (Right to Left)

   - We then scan backwards through the array
   - For each number, we check if it can form a ramp with any of our stored starting points
   - When we find a valid ramp, we calculate its width and update our answer if needed

   

Let's analyze the example array: `nums = [6,0,8,2,1,5]`

> Phase 1: Building the Stack
>
> ```
> Initial stack: empty
> Processing 6: stack = [(0,6)]         // index 0, value 6
> Processing 0: stack = [(0,6), (1,0)]  // 0 is smaller than 6, add it
> Processing 8: stack = [(0,6), (1,0)]  // 8 is larger, skip it
> Processing 2: stack = [(0,6), (1,0)]  // 2 is larger than 0, skip it
> Processing 1: stack = [(0,6), (1,0)]  // 1 is larger than 0, skip it
> Processing 5: stack = [(0,6), (1,0)]  // 5 is larger than 0, skip it
> ```
>
> *Note: for each pair in the stack, the first item is the index of that value.*
>
> Phase 2: Finding Maximum Widths
>
> ```
> Processing from right to left:
> 
> Start with stack = [(0,6), (1,0)]
> 
> 1. Process 5 (index 5):
>    - 5 > 0, so pop (1,0)
>    - Calculate width: 5 - 1 = 4
>    - Current max width = 4
>    - Stack = [(0,6)]
> 
> 2. Process 1 (index 4):
>    - 1 < 6, nothing to do
>    - Max width still 4
> 
> 3. Process 2 (index 3):
>    - 2 < 6, nothing to do
>    - Max width still 4
> 
> 4. Process 8 (index 2):
>    - 8 > 6, so pop (0,6)
>    - Calculate width: 2 - 0 = 2
>    - Max width still 4 (since 4 > 2)
>    - Stack is now empty
> 
> Final answer: 4
> ```
>
> 

The decreasing stack ensures we only keep track of potential starting points that could give us a maximum width and When we find a larger number while scanning from right to left, we know we've found the widest possible ramp for that starting point.

##### **Algorithm**

1. Initialize Variables
   - Create `ans = 0` to keep track of the widest ramp found
   - Create an empty `stack` to store potential starting points of ramps
   - The stack will store pairs of (index, value) for each candidate
2. Build the Monotonic Stack (Left to Right Scan)
   - For each index `i` in the array:
     - If the stack is empty OR the current value is less than the value at the top of the stack:
       - Add the pair `(i, nums[i])` to the stack
     - Otherwise:
       - Skip this value (it can't be a better starting point than what we already have)
3. **Find Maximum Width Ramps (Right to Left Scan)**
   - For each index `i` from the end of the array:
     - While the stack is not empty AND current value ≥ value at stack top:
       - Pop the top element from the stack
       - Calculate `width = current_index - popped_index`
       - Update `ans` if this width is larger
     - Continue until we either:
       - Empty the stack, or
       - Find a value in the stack that's larger than our current value
4. Return `ans`

##### **Implementation**

```python
class Solution:
    def maxWidthRamp(self, nums: List[int]) -> int:
        ans = 0
        stack = []
        
        for idx in range(len(nums)):
            if not stack or nums[idx] < stack[-1][1]:
                stack.append((idx, nums[idx]))

       
        for idx in range(len(nums) - 1, -1, -1):
            while stack and nums[idx] >= stack[-1][1]:
                ans = max(ans, idx - stack.pop()[0])

        return ans
```

##### **Time and Space Complexity Analysis**

The **Time complexity** of the program will be `O(n)` because we iterate through the array twice which is `O(2n)` and `O(2n) = O(n)`, where `n` is the length of the input array. While the **Space Complexity** will be `O(n)` because in the worse case, where our input already consists of already decreasing values, our stack size will be the same as the input size.
