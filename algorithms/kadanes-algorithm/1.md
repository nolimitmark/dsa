# KADANES'S ALGORITHM

Kadane's algorithm is a powerful technique used to solve one of the most common problems in programming: finding the sub-array with the maximum sum in a given array of integers. Whether you're working with positive, negative, or mixed numbers, this algorithm provides an efficient way to identify the solution with minimal effort. In this article, we'll explore the intuition behind Kadane's algorithm, walk through examples, and understand why it works.

------

Imagine you're given an array of integers and asked to find the sub-array with the highest sum. Let's start with an array of positive integers:

**Example:** `nums = [5, 2, 1, 4, 3]`

Here, the sub-array with the maximum sum is the entire array itself. Why? Because there are no negative numbers to decrease the total. Simple enough, right?

Now, let's add a twist. What if the array was modified to include some negative numbers, like this:

**Example:** `nums = [5, -2, 1, 4, -3]`

Can we still assume the sub-array with the maximum sum is the whole array? The answer is no, because negative numbers can reduce the overall sum, forcing us to carefully choose which sub-array to consider.

But how do we find this maximum sub-array? A straightforward approach is to generate all possible sub-arrays of `nums` and calculate their sums to determine which one is the largest. While this works, it's inefficient for larger arrays. Let's dive deeper into this approach before introducing a smarter solution.

1. `[5] = 5`
2. `[-2] = -2`
3. `[1] = 1`
4. `[4] = 4`
5. `[-3] = -3`
6. `[5,-2] = 3`
7. `[5,-2,1] = 4`
8. `[5,-2,1,4] = 8`
9. `[5,-2,1,4,-3] = 5`
10. `[-2,1] = -1`
11. `[-2,1,4] = 3`
12. `[-2,1,4,-3] = 0`
13. `[1,4] = 5`
14. `[1,4,-3] = 2`
15. `[4,-3] = 1`

Above are all sub-arrays of `nums`, and as we can see, the sub-array with the maximum sum is `[5, -2, 1, 4]` with a sum of `8`.

### Implementation

```python
def maxSubArray(nums):
    n = len(nums)

    sub_arrays = []

    # generate all sub-arrays
    for i in range(n):
        for j in range(i, n):
            sub_arrays.append(nums[i:j+1])
    
    max_subarray = []
    
    # look for the sub-array with maximum sum
    for arr in sub_arrays:
        if sum(arr) > sum(max_subarray):
            max_subarray = arr

    return max_subarray
```

Generating all sub-arrays and calculating their sums results in a time complexity of `O(n2)`, as we must examine every possible sub-array. This approach can be inefficient for larger arrays, can we do better? YES!.

### Optimizing with Kadane's Algorithm

Kadane's algorithm uses a dynamic programming approach to efficiently find the maximum sub-array sum. By scanning the array only once, Kadane's algorithm achieves a time complexity of `O(n)`.

This algorithm works seamlessly regardless of whether the array contains positive or negative numbers. The idea is to maintain two arrays:

1. `current_subarray`: Tracks the ongoing sub-array.
2. `max_subarray`: Tracks the best sub-array found so far.

### Algorithm:

1. Initialize: `current_subarray = []`, `max_subarray = [negative infinity]`
2. For each element `nums[i]`:
   - If adding `nums[i]` to `current_subarray` is greater than just `nums[i]`:
     - Extend `current_subarray` by including `nums[i]`.
   - Otherwise:
     - Start a new `current_subarray` with just `nums[i]`.
   - If the sum of `current_subarray` exceeds the sum of `max_subarray`:
     - Update `max_subarray` to match `current_subarray`.
3. After all iterations, `max_subarray` contains the desired sub-array.

#### Why does this work?

The idea is simple: we’re trying to find the best sub-array that gives the highest sum. To do this, we follow two basic rules:

1. Keep Going or Start Fresh
   - As we go through the array, we decide:
     - Should we add the current number to our ongoing sub-array?
     - Or should we start a new sub-array with just the current number?
   - If adding the current number makes things better, we add it. Otherwise, we start fresh.
2. Always Remember the Best
   - Every time our ongoing sub-array has a sum greater than the best one we’ve seen so far, we update our "best" sub-array.

By the end, the "best" sub-array will have the highest possible sum.

### Implementation

```python
def maxSubArray(nums):
    n = len(nums)

    current_subarray, max_subarray = [], [float('-inf')]

    for i in range(n):
        if nums[i] + sum(current_subarray) >= nums[i]:
            current_subarray.append(nums[i])
        else:
            current_subarray = [nums[i]]
        
        # update max_subarray if neceassary
        if sum(current_subarray) > sum(max_subarray):
            max_subarray = current_subarray.copy()
    
    return max_subarray
```

We have implemented the solution where we maximum sub-array it self, Now, let's adjust our code to just return it's sum.

```python
def maxSubArraySum(nums):
    n = len(nums)

    current_sum, max_sum = 0, float('-inf')

    for i in range(n):
        current_sum = max(nums[i] + current_sum, nums[i])
        max_sum = max(max_sum, current_sum)

    return max_sum
```

### Complexity Analysis

- **Time Complexity**: Regardless of whether we track the maximum sub-array or just the maximum sum, the algorithm only requires a single pass through the array. Thus, the time complexity is O(n)O(n)O(n).
- **Space Complexity**: If we only track the current sum and the maximum sum, the space complexity is `O(1)`, as we only use a few variables for calculations. Otherwise, If we explicitly track the current and maximum sub-arrays, the space complexity can grow to `O(n)`. In the worst case, where all numbers are positive, the current and maximum sub-arrays could be as large as the input array itself.

Note: Kadane's algorithm is not only used for the maximum sub-array problem. It can be applied to other problems where you need to find optimal sub-arrays.
