

# **Overview of the Two Pointer technique**

The two-pointer technique is a straightforward approach often used to traverse linear data structures efficiently to find a solution to a given problem. The idea is to use two pointers, typically starting at different ends of the array, and adjust their positions based on certain criteria until a condition is met or a base case is reached.

This technique is commonly applied in problems like the iterative version of binary search and the sliding window technique. It is considered an efficient approach because it reduces the need for nested iterations, leading to improved time complexity.

By using the two-pointer method, certain problems that would typically take `O(n^2)` with brute force can be solved in `O(n)` or `O(n log n)` time, making it a powerful tool in algorithm design.

## **Example Demonstrating the Two Pointer technique**

Consider a basic example where we have a sorted array of integers `nums` with the values `[1, 3, 4, 7]`.

Suppose we want to select two indices `i` and `j` from `nums` such that the absolute difference between `nums[i]` and `nums[j]` is minimized. Using a brute force approach, it would involve trying every possible pair of indices `(i, j)` and tracking the smallest absolute difference encountered.

Now the two-pointer technique can come into play here. Notice that since the array is sorted, we only need to consider adjacent elements to find the minimum absolute difference. Why? Because in a sorted array, the smallest difference between any two numbers will always occur between consecutive elements. This eliminates the need for nested loops and reduces the time complexity to O(n).

**Algorithm**

1. Initialize a variable `md` to infinity. This will hold the minimum absolute difference found during the iteration.
2. Set up two pointers, `left` and `right`, initialized to `0` and `1` respectively, to track the positions in the array.
3. Begin iterating as long as `right` is less than the length of the array.
4. In each iteration, calculate the absolute difference `absDiff` between the elements at `nums[left]` and `nums[right]`.
5. Update `md` to be the smaller of its current value or `absDiff`.
6. Increment the `left` and `right` pointers by 1.
7. After the loop, return the value of `md`.

**Implementation**

```python
def minimumAbsoluteDifference(nums):
    md = float('inf')
    
	left, right = 0, 1
    
    while right < len(nums):
        absDiff = abs(nums[left] - nums[right])
        md = min(md, absDiff)
        
        left, right = left + 1, right + 1
    return md
```

The Time Complexity of the program is `O(n)` where `n` is the length of the array because we only scan through the array once, while the Space Complexity will be `O(1)` as we are only using a constant amount of additional space.

**Note**: The initial placement of the two pointers isn't strictly fixed at the beginning of the array. Depending on the specific problem, they could start from any two points in the array, such as both ends or other positions, to suit the context.

Let’s consider another example using the same array `nums = [1, 3, 4, 7]`, but this time we aim to find two indices `i` and `j` such that the sum of `nums[i] + nums[j]` is equal to a given `target`. Our goal is to return the pair of indices of any valid pair.

In a brute force approach, we would check all possible pairs `(i, j)` to find if their sum equals the `target`. But this approach would be inefficient, requiring `O(n^2)` time complexity.

Using the two-pointer technique, we can solve this more efficiently. Here’s how:

1. Initialize two pointers: `left` starting at index `0` (the beginning of the array), and `right` starting at the last index (`len(nums) - 1`).
2. Calculate the sum of `nums[left] + nums[right]`.
3. If the sum equals the `target`, return the indices `left` and `right` as the solution.
4. If the sum is greater than `target`, it means we need to decrease the sum, so we decrement the `right` pointer by 1.
5. If the sum is less than `target`, we need to increase the sum, so we increment the `left` pointer by 1.
6. Continue adjusting the pointers until a valid pair is found or the pointers cross each other.

**Implementation**

```python
def findPair(nums, target):
    left, right = 0, len(nums) - 1
    
    while left < right:
        current_sum = nums[left] + nums[right]
        
        if current_sum == target:
            return (left, right)
        elif current_sum < target:
            left += 1
        else:
            right -= 1
    
    return None  # Return None if no solution is found

```

This approach reduces the time complexity to `O(n)` since we are only traversing the array once with the two pointers. 

## **Advantages of the Two-Pointer Technique**

1. It significantly reduces time complexity compared to brute force solutions.
2. It often requires minimal extra space, making it memory-efficient.
3. It is useful for a variety of tasks such as sorting, searching, and pair-sum problems.
4. The logic is straightforward, making it easy to implement and debug.
5. It works exceptionally well with sorted arrays or data.

## **Disadvantages of the Two-Pointer Technique**

1. It's not suitable for all problems, especially when data is unsorted or not easily comparable.
2. It may require pre-processing, such as sorting, which adds to the overall complexity.
3. It's not effective for problems that require more than two pointers or complex navigation.
4. Careful handling is needed to avoid issues with overlapping pointers or out-of-bound errors.
5. It cannot be applied universally and needs tailored logic for different problems.
